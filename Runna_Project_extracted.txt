Chapter  One  |  Introduction  and  Rationale  1.1  Motivation  
Running  is  one  of  the  most  accessible  and  widely  adopted  forms  of  physical  activity,  requiring  
minimal
 
equipment
 
while
 
providing
 
significant
 
benefits
 
to
 
both
 
physical
 
and
 
mental
 
health.
 
With
 
the
 
rapid
 
advancement
 
of
 
mobile
 
technology,
 
modern
 
running
 
applications
 
have
 
evolved
 
to
 
include
 
features
 
such
 
as
 
GPS
 
tracking,
 
pace
 
analysis,
 
and
 
performance
 
statistics.
 
However,
 
despite
 
these
 
improvements,
 
existing
 
solutions
 
remain
 
largely
 
performance-centric
 
and
 
fail
 
to
 
address
 
several
 
critical
 
gaps
 
in
 
user
 
experience,
 
safety,
 
and
 
social
 
interaction.
 
First,  current  applications  lack  proactive  safety  intelligence.  While  runners,  especially  those  who  
run
 
alone
 
or
 
in
 
unfamiliar
 
environments
 
face
 
risks
 
such
 
as
 
falls,
 
collisions,
 
or
 
sudden
 
health
 
incidents,
 
most
 
existing
 
platforms
 
provide
 
only
 
passive
 
safety
 
features,
 
such
 
as
 
manual
 
location
 
sharing.
 
There
 
is
 
little
 
to
 
no
 
real-time
 
analysis
 
of
 
motion
 
data
 
to
 
detect
 
abnormal
 
events
 
and
 
trigger
 
automated
 
emergency
 
responses.
 
This
 
limitation
 
exposes
 
users
 
to
 
significant
 
risk
 
and
 
reduces
 
trust
 
in
 
the
 
system
 
during
 
critical
 
situations.
 
Second,  route  planning  remains  inefficient  and  non-intuitive.  Most  applications  require  users  to  
manually
 
select
 
routes
 
or
 
rely
 
on
 
static,
 
predefined
 
paths.
 
These
 
approaches
 
do
 
not
 
adapt
 
to
 
user
 
preferences
 
such
 
as
 
desired
 
distance,
 
environmental
 
conditions
 
(e.g.,
 
parks,
 
low
 
traffic
 
areas),
 
or
 
route
 
structure
 
(e.g.,
 
loop
 
vs.
 
out-and-back).
 
As
 
a
 
result,
 
users
 
must
 
invest
 
additional
 
time
 
and
 
effort
 
in
 
planning,
 
which
 
negatively
 
impacts
 
usability
 
and
 
discourages
 
consistent
 
engagement.
 
Third,  social  interaction  in  existing  running  platforms  is  limited  and  often  indirect.  While  some  
applications
 
include
 
community
 
features,
 
they
 
are
 
typically
 
asynchronous
 
(e.g.,
 
activity
 
sharing,
 
comments,
 
likes)
 
and
 
do
 
not
 
effectively
 
support
 
users
 
in
 
finding
 
running
 
partners
 
in
 
a
 
simple
 
and
 
practical
 
way.
 
Moreover,
 
real-time
 
matching
 
systems,
 
while
 
powerful,
 
introduce
 
complexity,
 
privacy
 
concerns,
 
and
 
scalability
 
challenges.
 
To  address  these  limitations,  Runna  is  proposed  as  an  intelligent  and  user-centric  running  assistant  
platform.

1.2  Purpose  
The  purpose  of  Runna  is  to  develop  an  intelligent  mobile  application  that  enhances  the  running  
experience
 
through
 
the
 
integration
 
of
 
artificial
 
intelligence,
 
safety
 
monitoring,
 
and
 
social
 
interaction
 
within
 
a
 
unified
 
platform.
 
The  system  is  designed  around  three  core  components:  
1.  AI-Based  Route  Generation  
Runna  enables  users  to  generate  personalized  running  routes  through  natural  language  input,  
allowing
 
them
 
to
 
describe
 
their
 
requirements
 
in
 
a
 
simple
 
and
 
intuitive
 
way
 
(e.g.,
 
“5
 
km
 
park
 
route
 
with
 
low
 
traffic”).
 
The  system  transforms  this  input  into  structured  parameters,  including:  
●  Target  distance  ●  Route  type  (loop  /  out-and-back)  ●  Environmental  preferences  (park,  urban,  low  traffic)  
These  parameters  are  processed  using:  
●  Natural  Language  Processing  (NLP)  ●  Map-based  graph  traversal  algorithms  ●  Heuristic  optimization  techniques  
The  system  then  generates  and  evaluates  multiple  route  candidates  and  selects  the  optimal  route  
based
 
on
 
criteria
 
such
 
as
 
distance
 
accuracy,
 
safety,
 
and
 
usability.

2.  Social  Feed  and  Chat  System  
Instead  of  complex  real-time  matching,  Runna  provides  a  simplified  and  scalable  social  interaction  
model
 
through
 
a
 
post-based
 
feed
 
and
 
private
 
messaging
 
system.
 
Users  can:  
●  Create  posts  to  find  running  partners  ●  Tag  posts  with  a  specific  province  ●  Search  posts  using  keywords  ●  Filter  content  based  on  location  (province)  ●  Communicate  via  private  chat  
This  approach:  
●  Reduces  system  complexity  ●  Enhances  user  privacy  (no  real-time  location  sharing)  ●  Enables  organic  and  flexible  social  interaction  
3.  Real-Time  Accident  Detection  
Runna  integrates  a  sensor-based  safety  monitoring  system  that  continuously  analyzes  motion  data  
from
 
smartphone
 
sensors,
 
including:
 
●  Accelerometer  ●  Gyroscope  ●  GPS  
The  system  applies  sensor  fusion  techniques  and  multi-condition  validation  to  detect  abnormal  
patterns
 
such
 
as:
 
●  High-impact  collisions  ●  Falls  with  inactivity  ●  Sudden  abnormal  velocity  changes  
When  an  anomaly  is  detected,  the  system:  
●  Triggers  an  alert  ●  Initiates  a  countdown  mechanism  ●  Sends  emergency  notifications  with  location  data  if  no  response  is  received   Document  Name

1.3  Target  users  
Independent  runners  who  require  safety  assurance  during  solo  activities  
Beginner  runners  who  need  guidance  in  route  planning  and  user-friendly  tools  
Social  runners  who  want  to  find  running  partners  through  simple  and  flexible  interaction  
Safety-focused  users  who  prioritize  emergency  detection  and  monitoring  during  outdoor  activities  
 
1.4  Project  Scope  
The  Runna  project  focuses  on  the  development  of  three  primary  subsystems:  
1.  AI  Route  Generation  Engine  
●  Natural  language  processing  for  user  input  interpretation  ●  Route  generation  using  map  APIs  ●  Optimization  based  on  distance,  safety,  and  environment  
2.  Social  Feed  &  Chat  System  
●  Post  creation  and  management  ●  Province-based  tagging  and  filtering  ●  Keyword-based  search  functionality  ●  Private  messaging  system  between  users  
3.  Sensor-Based  Accident  Detection  System  
●  Real-time  sensor  data  collection  ●  Motion  analysis  using  sensor  fusion  ●  Event  detection  and  classification  ●  Emergency  alert  and  notification  system

2.2.2.5  Gemini  API  
 
Gemini  is  used  for  Natural  Language  Processing  (NLP)  to  interpret  user  input  and  convert  it  into  
structured
 
route
 
constraints.
 
Core  Functionality:
